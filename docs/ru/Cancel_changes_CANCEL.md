---
title: 'Отмена изменений (CANCEL)'
---

Оператор *отмены изменений* полностью очищает текущую [сессию изменений](Change_sessions.md).

Очистка сессии возвращает все накопленные в ней изменения: значения [свойств данных](Data_properties_DATA.md) (в том числе [локальных](Data_properties_DATA.md#local)) возвращаются к тем, что были в начале сессии, [созданные](New_object_NEW.md) в сессии объекты исчезают, [изменения классов](Class_change_CHANGECLASS_DELETE.md) у существующих объектов откатываются.

Как и для других операторов управления сессиями, для оператора отмены изменений можно явно указать [вложенные локальные свойства](Session_management.md#nested).

:::warning
Этот оператор работает по-другому, если выполняется во время [транзакции применения](Apply_changes_APPLY.md) — например, внутри применяемого действия или внутри обработчика [глобального синхронного события](Events.md#change). В этом случае он не очищает сессию, а отменяет идущее в данный момент применение.
:::

### Язык

Для объявления действия, реализующего отмену изменений, используется [оператор `CANCEL`](CANCEL_operator.md).

### Примеры

```lsf
CLASS Sku;
in = DATA LOCAL BOOLEAN (Sku);

dropChanges()  {
    CANCEL NESTED (in[Sku]); // отменяем все изменения, кроме свойства `in[Sku]`
}
```
