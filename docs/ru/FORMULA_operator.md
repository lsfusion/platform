Оператор `FORMULA` - создание [свойства](Properties.md), реализующего [пользовательскую формулу](Custom_formula_FORMULA.md).

### Синтаксис

```
FORMULA [NULL] [className [valueId]] [syntaxType1] text1, ..., [syntaxTypeN] textN [(classId1 [paramId1], ..., classIdK [paramIdK])] [NULL]
```

### Описание

Оператор `FORMULA` создает свойство, выполняющее произвольную формулу на языке SQL. При этом есть возможность задать формулу отдельно для различных диалектов SQL, для того, чтобы эти свойства были переносимы между различными СУБД.

`FORMULA` — [контекстно-независимый](Property_operators.md#contextindependent) оператор-свойство: он не может стоять внутри [выражений](Expression.md). Использовать его можно в правой части `=`-объявления или анонимно внутри скобок в применении [оператора `JOIN`](JOIN_operator.md).

#### Два режима работы

Оператор FORMULA работает в одном из двух режимов, который выбирается автоматически в зависимости от того, упоминаются ли все параметры свойства в тексте формулы или нет:

**1. Режим выражения** (по умолчанию)

Все параметры упоминаются в тексте формулы. Для каждой комбинации входных значений формула возвращает ровно один результат — как математическая функция. Это самый распространённый вариант использования.

```lsf
round(number, digits) = FORMULA 'round(CAST(($1) as numeric), $2)';
```

**2. Режим табличной функции**

Некоторые функции баз данных возвращают не одно значение, а *набор строк*. Например, «разбить строку по разделителю» может вернуть ноль, одну или несколько частей. В lsFusion свойство должно возвращать ровно одно значение для каждой уникальной комбинации параметров — поэтому нужен дополнительный параметр, позволяющий указать конкретную строку из результата.

Когда хотя бы один параметр свойства **отсутствует** в тексте формулы, lsFusion автоматически переключается в режим табличной функции. Неиспользованный(е) параметр(ы) становятся ключевыми измерениями, однозначно идентифицирующими каждую строку результата. Токен `valueName` задаёт, из какого столбца вывода функции берётся итоговое значение свойства.

Два режима взаимно исключают друг друга и выбираются автоматически — никаких дополнительных ключевых слов не нужно.

##### Как неиспользованные параметры идентифицируют строку

Есть два варианта:

**`row INTEGER` — доступ по порядковому номеру**

Если ровно один неиспользованный параметр называется в точности `row` и имеет тип `INTEGER`, lsFusion автоматически присваивает строкам последовательные целые числа (1, 2, 3, …). Чтобы получить третий элемент, нужно передать `row = 3`; чтобы обойти все элементы — использовать агрегацию по `row`.

```lsf
// элементы JSON-массива, доступ по индексу начиная с 1
array(JSON json, INTEGER row) = FORMULA JSON value 'jsonb_array_elements($json)';

// слова строки, разбитой по разделителю, доступ по индексу
array(STRING string, STRING delimeter, INTEGER row) = FORMULA STRING value 'unnest(string_to_array($string, $delimeter))';
```

**Любое другое имя/тип — доступ по ключевому столбцу**

Во всех остальных случаях имя каждого неиспользованного параметра должно совпадать с именем столбца, который функция сама возвращает в своём выводе. lsFusion использует эти столбцы как ключи: чтобы получить нужную строку, нужно передать соответствующие значения ключей как параметры. Несколько неиспользованных параметров образуют составной ключ — каждый независимо сопоставляется с одноимённым столбцом вывода.

Функция должна естественным образом возвращать столбцы с теми же именами, что и параметры. Например, `jsonb_each` возвращает пары `(key, value)` — поэтому неиспользованный параметр `key STRING` напрямую сопоставляется со столбцом `key`, а `valueName = value` задаёт, какой столбец читать как значение.

```lsf
// значение по заданному ключу внутри JSON-объекта
map(JSON json, STRING key) = FORMULA JSON value 'jsonb_each($json)';

// то же, но значение как обычная строка
mapText(JSON json, STRING key) = FORMULA STRING value 'jsonb_each_text($json)';
```

#### Ссылки на параметры в тексте формулы

Параметры можно упоминать либо **по позиции** (`$1`, `$2`, ...), либо **по имени** (`$paramName`).

Именованные ссылки разрешаются в следующем порядке:
1. Если после текста формулы задан явный список параметров `(...)`, используются имена из него.
2. Иначе имена берутся из списка параметров самого свойства.

Позиционные и именованные ссылки можно свободно смешивать.

```lsf
// имена берутся из параметров свойства
substring(STRING string, STRING regexp) = FORMULA NULL STRING PG 'SELECT substring($string, $regexp)';

// имена объявлены явно в (...)
toDateTime = FORMULA DATETIME '($date) + ($time)' (DATE date, TIME time);
```

#### Семантика NULL

По умолчанию, если хотя бы одно из входных значений равно NULL, формула сразу возвращает NULL, не вычисляя выражение (стандартное арифметическое поведение: `NULL + 5 = NULL`).

Два ключевых слова `NULL` изменяют это поведение (см. раздел «Параметры»). Они независимы и могут быть заданы одновременно.

### Параметры

- `NULL`

    Ключевое слово, которое ослабляет правила обработки `NULL` по умолчанию у нетабличной формулы (в табличном режиме обе позиции игнорируются). Две формы по нарастанию силы:
    - перед `className` — объявляет, что свойство может возвращать `NULL`, даже если все значения параметров не `NULL`. Без этой формы (и без завершающей ниже) формула обязана для не-`NULL` аргументов всегда возвращать не-`NULL` значение — невыполнение этого условия может привести к непредсказуемым результатам.
    - в конце (после опционального списка параметров) — формула принимает `NULL`-значения параметров и выполняется над ними; без него `NULL`-аргумент сразу даёт `NULL`-результат без обращения к формуле. Эта форма поглощает начальную — раз формула сама видит `NULL`-аргументы и решает, что вернуть, отдельное объявление о возможности `NULL`-результата ничего не добавляет.

- `className`

    Имя [встроенного класса](Built-in_classes.md) возвращаемого свойством значения. Если не задано, результирующим классом считается [общий предок](Built-in_classes.md#commonparentclass) всех операндов свойства. Обязателен для табличных формул (см. список параметров ниже): колонке-значению возвращаемой таблицы нужен явный тип.

- `valueId`

    Идентификатор или [строковый литерал](IDs.md#strliteral) сразу после `className`. Задаёт имя колонки-значения в таблице, возвращаемой формулой — обязателен для табличных формул (см. список параметров ниже) и игнорируется для нетабличных. Форма со строковым литералом обычно используется для указания существующей колонки во внешней таблице; форма с идентификатором — для значения, возвращаемого set-returning функцией.

- `syntaxType1, ..., syntaxTypeN`

    Ключевые слова, определяющие типы диалектов SQL. На данный момент поддерживаются следующие типы:

    - `PG` - синтаксис PostgreSQL
    - `MS` - синтаксис MS SQL Server

  Если тип диалекта не задан явно, то соответствующий текст формулы устанавливается, как текст по умолчанию. Каждый из типов (или отсутствие типа) должен встречаться в операторе не более одного раза.

- `text1, ..., textN`

    [Строковые литералы](IDs.md#strliteral), каждый из которых содержит формулу в SQL-синтаксисе. Параметры формулы подставляются как `$1`, `$2`, ... по позиции либо как `$paramId`, если у параметра есть имя. Нумерация позиционных параметров начинается с `1`. Если задано несколько реализаций (по одной на диалект плюс опционально умолчательная), все они должны описывать один и тот же набор параметров — несовпадение арности между ними отклоняется на этапе разбора.

- `classId1 [paramId1], ..., classIdK [paramIdK]`

    Опциональное объявление классов параметров формулы (и, для каждого, опционального имени, которое можно использовать внутри формулы как `$paramId`). Если не указано, классы и имена параметров берутся из внешнего объявления свойства. Арность результирующего свойства равна большему из `K` и максимального позиционного индекса, на который ссылается текст формулы; позиционные обращения за пределы `K` молча расширяют список параметров автогенерируемыми именами.

    Используемые в тексте формулы параметры передаются как входы в SQL-выражение в момент вычисления; неиспользуемые становятся ключевыми колонками возвращаемой таблицы (см. `valueId` выше для колонки-значения) и должны совпадать по именам с колонками, которые возвращает SQL-выражение. Единственный неиспользуемый параметр класса `INTEGER` с именем `row` — отдельный случай: платформа подставляет для него номера строк через `ROW_NUMBER() OVER ()`, поэтому такая идиома работает с любым табличным SQL-выражением.

### Примеры

```lsf
// свойство с двумя параметрами: округляемым числом и количеством знаков после запятой
round(number, digits) = FORMULA 'round(CAST(($1) as numeric),$2)';

// свойство, преобразовывающее значение, переданное в качестве аргумента, к строковому типу длиной в 15 символов.
toString15(str) = FORMULA BPSTRING[15] 'CAST($1 AS character(15))';

// свойство с двумя различными реализациями для разных диалектов SQL
jumpWorkdays = FORMULA NULL DATE PG 'jumpWorkdays($1, $2, $3)', MS 'dbo.jumpWorkdays($1, $2, $3)';

// обращение к параметрам по имени: $json / $field берутся из внешней сигнатуры
field (JSON json, STRING field) = FORMULA JSON 'jsonb_extract_path($json, $field)';

// явный список параметров с именами для $date / $time
toDateTime = FORMULA DATETIME '$date + $time' (DATE date, TIME time);

// табличная формула: `row` не используется в тексте, поэтому становится ключевой колонкой
// таблицы, возвращаемой jsonb_array_elements; `value` задаёт имя колонки-значения
array (JSON json, INTEGER row) = FORMULA JSON value 'jsonb_array_elements($json)';

// табличная формула с именованной ключевой колонкой: jsonb_each возвращает строки (key, value);
// `key` (неиспользуемый параметр) и `value` (valueId) совпадают с именами колонок этой записи
map (JSON json, STRING key) = FORMULA JSON value 'jsonb_each($json)';

// подключение внешней таблицы: текст формулы — имя таблицы, строковый литерал после
// STRING задаёт имя колонки-значения, а `key0` — ключевая колонка этой внешней таблицы
country (LONG key0) = FORMULA STRING 'country_name' '_auto_country';

// завершающее NULL: формула сама решает, что вернуть, когда аргумент равен NULL
coalesceName = FORMULA TEXT 'COALESCE($1, \'(unknown)\')' (TEXT name) NULL;
```
