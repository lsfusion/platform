Оператор *ввода значения* создаёт [действие](Actions.md), которое запрашивает у пользователя [ввод значения](Value_input.md). Запрашиваемое значение может быть как объектом [встроенного класса](Built-in_classes.md) (например, строкой, числом, датой или файлом), так и объектом пользовательского класса. Пользователь при желании может [отменить ввод](Value_input.md#result), например, нажав на клавиатуре клавишу `Esc`.

Этот оператор можно использовать только в обработке [событий изменения](Form_events.md#property) свойства на форме.

### Запрашиваемое значение {#value}

Если запрашивается объект встроенного класса, платформа показывает пользователю [элемент ввода](Value_input.md#execution), соответствующий этому классу, и пользователь вводит его напрямую.

Если запрашивается объект пользовательского класса, пользователь не вводит его напрямую, а выбирает из [формы выбора](Interactive_view.md#edtClass) объектов класса.

### Предлагаемые значения {#list}

Значения, предлагаемые пользователю для выбора, можно ограничить заданным набором, а этот набор дополнительно сузить условием. Для встроенного класса такой набор показывается пользователю как список подсказок при вводе; для пользовательского класса он ограничивает объекты, доступные для выбора.

Кроме того, если ввод используется для [изменения](Value_input.md#initial) свойства, предлагаемые значения по умолчанию дополнительно ограничиваются теми, выбор которых не нарушит существующих в системе [ограничений](Constraints.md) — так же, как в [диалоговой форме](In_an_interactive_view_SHOW_DIALOG.md#dialog) открытия формы. При необходимости это ограничение можно отключить.

### Общие возможности ввода

Также как и в остальных операторах ввода значения, в этом операторе можно:

-   задавать [начальные значения](Value_input.md#initial) объектов
-   задавать [основное и альтернативное](Value_input.md#result) действия. Первое вызовется, если ввод был успешно завершён, второе - в обратном случае (если ввод был отменён).
-   [осуществлять изменение](Value_input.md#initial) заданного свойства

### Язык

Синтаксис оператора ввода значения описывается [оператором `INPUT`](../language/INPUT_operator.md).

### Примеры

```lsf
changeCustomer (Order o)  {
    INPUT s = STRING[100] DO {
        customer(o) <- s;
        IF s THEN
            MESSAGE 'Customer changed to ' + s;
        ELSE
            MESSAGE 'Customer dropped';
    }
}

FORM order
    OBJECTS o = Order
    PROPERTIES(o) customer ON CHANGE changeCustomer(o)
;

testFile  {
    INPUT f = FILE DO { // запрашиваем диалог по выбору файла
        open(f); // открываем выбранный файл
    }
}

sku = DATA Sku (OrderDetail);

// выбор объекта пользовательского класса: предлагаем sku по их наименованию и оставляем только те, что в наличии
changeSku (OrderDetail d)  {
    INPUT s = sku(d) CHANGE LIST name(s) WHERE inStock(s);
}
```
