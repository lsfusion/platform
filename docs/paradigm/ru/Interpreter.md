Интерпретатор — это интерактивный механизм, позволяющий администратору выполнять произвольный код на языке lsFusion (или вызов кода на Java) на уже работающем сервере приложений, без перезапуска и пересборки проекта. Код вводится в форме `Система > Интерпретатор` (`System.interpreter` в навигаторе) и выполняется на сервере в рамках новой сессии.

Любой код, выполняемый через интерпретатор, имеет тот же доступ к элементам системы (классам, свойствам, действиям, формам), что и обычный модуль, загруженный при запуске. Это позволяет проверять гипотезы, читать и точечно менять данные, запускать существующие действия и формы и временно вводить новые расчёты — без изменения файлов проекта.

### Режимы ввода

Интерпретатор принимает четыре формы исходного кода — выбор режима определяет, как введённый текст оборачивается перед выполнением:

|Режим| Что вводится                                                                           |Что запускается|
|---|----------------------------------------------------------------------------------------|---|
|`Скрипт`| Последовательность инструкций модуля, в которой объявлено действие с именем `run`.     |Запускается это действие `run`.|
|`Действие`| Тело действия — последовательность операторов действий и объявлений локальных свойств. |Платформа оборачивает тело в действие `run` и запускает его.|
|`Форма`| Объявление формы.                                                                      |Платформа создаёт форму с этим объявлением и открывает её [в интерактивном представлении](In_an_interactive_view_SHOW_DIALOG.md).|
|`Java`| Фрагмент кода на Java.                                                                 |Платформа оборачивает фрагмент во [внутренний вызов](Internal_call_INTERNAL.md) Java и запускает его.|

### Выполнение

Введённый код запускается так же, как [оператор выполнения программного кода (`EVAL`)](Eval_EVAL.md): с полной видимостью всех загруженных модулей проекта и в рамках новой [сессии](Change_sessions.md). Чтобы изменения данных сохранились в базе, код должен сам выполнить [применение изменений](Apply_changes_APPLY.md).

Если при выполнении возникает исключение, платформа открывает форму с текстом исключения и стэками вызовов на стороне Java и на стороне lsFusion; с этой формы можно скопировать всю эту информацию в буфер обмена.

### Журнал

Каждый запуск через интерпретатор сохраняется в журнал скриптов (форма `Журнал скриптов`) с автором, текстом скрипта, временем последнего изменения текста и временем последнего запуска. Сохранённый скрипт можно вернуть в интерпретатор кнопкой `Найти скрипт`.

### Временные данные

Форма интерпретатора содержит таблицу со строковыми, числовыми и датовыми колонками (`string1`..`string10`, `numeric1`..`numeric5`, `date1`..`date3`), к объектам которой выполняемый код может обращаться напрямую. Это позволяет вручную задать входные данные для скрипта и сразу увидеть результат в той же форме.

### Доступ

Доступ к интерпретатору регулируется [политикой безопасности](Security_policy.md): достаточно дать роли доступ к навигаторной форме `System.interpreter`. Пользователю, имеющему доступ к этой форме, также разрешён внешний доступ к [программному интерфейсу](Access_from_an_external_system.md) даже при отключённом параметре работы `enableAPI` (см. [параметры работы](Working_parameters.md)), поскольку этот пользователь и без того может выполнить любой код через интерпретатор.
