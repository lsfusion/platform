*Монитор процессов* — встроенный механизм платформы, показывающий состояние сервера приложений и сервера БД в текущий момент времени: активные потоки сервера приложений, активные SQL-запросы, цепочки блокировок, а также набор действий для прерывания и принудительного завершения этих процессов.

### Содержимое монитора

Монитор открывается формой `Администрирование > Система > Монитор процессов` и состоит из трёх вкладок:

|Вкладка|Что показывает|
|---|---|
|Все процессы|Полный список активных серверных потоков и активных SQL-запросов, объединённых по логическим процессам|
|Блокирующие процессы|Процессы, удерживающие блокировки, которых ждут другие процессы|
|Блокируемые процессы|Процессы, ожидающие освобождения блокировок|

Каждая запись объединяет два аспекта одного логического процесса: серверный поток (Java) и связанный с ним SQL-запрос (если процесс в этот момент работает с БД).

### Серверные потоки

Для каждого активного потока сервера приложений отображаются: идентификатор и имя потока, статус (`RUNNABLE`, `BLOCKED`, `WAITING`, `TIMED_WAITING`), объём выделенной потоку памяти, информация об удерживаемой им блокировке и стек вызовов (Java-стек на момент опроса). Простаивающие системные потоки — поток событий, сетевой ввод-вывод, служебные потоки JVM — из списка отфильтровываются как неинформативные.

### SQL-запросы

Для каждого активного SQL-запроса отображаются: идентификатор процесса БД, имя пользователя, имя клиентского компьютера, состояние транзакции (активна, ожидает, idle), полный текст запроса и его сокращённое представление, события ожидания и стек SQL-вызовов.

Запрос подсвечивается фоном по возрасту транзакции, в которой он выполняется: чем дольше транзакция удерживает изменения, тем агрессивнее фон, что позволяет визуально отделить кратковременные операции от долгих или зависших.

### Блокировки

Для каждого процесса дополнительно показывается список блокируемых им процессов и список процессов, которые блокируют его самого. На вкладках *Блокирующие* и *Блокируемые* эта связь раскрывается в иерархию: можно перейти от удерживающего блокировку процесса к ожидающим его, и наоборот. Граф блокировок строится рекурсивно — циклические взаимные блокировки (deadlock) обнаруживаются и помечаются отдельно.

### Прерывание и принудительное завершение

Для каждого процесса в мониторе доступны четыре действия:

|Действие|Что делает|Когда применять|
|---|---|---|
|Отменить Java-процесс|Посылает потоку запрос на отмену через стандартный механизм прерывания; процесс завершает текущую операцию и выходит при ближайшей проверке флага прерывания|Штатная отмена долгого или зависшего пользовательского действия|
|Снять Java-процесс|Принудительное завершение потока|Крайний случай: процесс не реагирует на отмену|
|Отменить SQL-процесс|Посылает серверу БД запрос на отмену выполняемого запроса|Долгий или зависший запрос, после которого процесс должен продолжить штатно|
|Снять SQL-процесс|Принудительное завершение процесса БД|Зависший процесс БД, мешающий другим|

«Снять» — операция последней инстанции: после неё состояние сервера приложений и сервера БД может потребовать перезапуска.

### Фильтр по типу процессов

Монитор показывает процессы через один из четырёх фильтров: все процессы, все активные процессы, только активные SQL-запросы или только активные потоки сервера приложений. «Активные» фильтры отсеивают процессы, простаивающие вне транзакции или запроса, и оставляют только те, что действительно нагружают сервер на момент опроса. По умолчанию монитор открывается на активных SQL-запросах.
