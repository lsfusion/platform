*Системный модуль* — [модуль](Modules.md), поставляемый вместе с платформой. Прикладной проект подключает такие модули через `REQUIRE` и пользуется их объявлениями (классами, свойствами, действиями, формами) как готовой стандартной библиотекой; платформа никогда не ожидает их переопределения.

Системные модули живут под `server/src/main/lsfusion/` в репозитории платформы: ядро — в подпапке `system/`, прикладные дополнения — в `utils/`. Модуль `System` подключается к любой проектной сборке автоматически, остальные требуют явного `REQUIRE`.

### Состав

#### Ядро (`system/`)

| Модуль          | Назначение |
|-----------------|-----------|
| [`System`](System_System.md) | Корневые типы, базовые классы и инфраструктура. Автоматически подключается ко всем модулям |
| [`Time`](System_Time.md) | Свойства и операции над датой и временем |
| [`Authentication`](System_Authentication.md) | Учётные записи, контакты, вход в систему, локаль, пароли |
| [`Security`](System_Security.md) | Роли и политики доступа |
| [`Service`](System_Service.md) | Сервисные действия, мониторинг БД, перерасчёт агрегаций, настройки |
| [`SystemEvents`](System_SystemEvents.md) | События жизненного цикла сервера, подключения, исключения, оформление |
| [`UserEvents`](System_UserEvents.md) | Программный доступ к фильтрам и сортировкам формы |
| [`Reflection`](System_Reflection.md) | Метаданные о навигаторе, формах, свойствах, таблицах |
| [`Scheduler`](Scheduler.md) | Запуск действий по расписанию |
| [`Email`](System_Email.md) | Учётные записи почты, отправка и приём писем |
| [`Icon`](System_Icon.md) | Каталог иконок интерфейса и их поиск |
| [`Utils`](System_Utils.md) | Сборник вспомогательных свойств и действий: файловая система, кодирование, строки, числа, JSON, полнотекстовый поиск и т. п. |

#### Прикладные дополнения (`utils/`)

| Модуль            | Назначение                                                                 |
|-------------------|----------------------------------------------------------------------------|
| `Backup`          | Снятие и восстановление резервных копий БД (см. [Резервное копирование](Backup_restore.md)) |
| `Chat`            | Внутренний чат (см. [Чат](Chat.md))                                        |
| `Eval`            | Динамическое выполнение lsFusion-кода (см. [`EVAL`](Eval_EVAL.md))         |
| `Excel`           | Работа с XLS / XLSX-файлами                                                 |
| `Document` / `Word` | Печатные документы и шаблоны                                              |
| `Image` / `OpenCV`| Обработка изображений                                                       |
| `I18n`            | Машинный перевод строк                                                     |
| `Integration`     | Промежуточные интеграционные действия                                       |
| `MasterData`      | Базовые шаблоны для справочников                                            |
| [`Numerator`](Utils_Numerator.md) | Универсальные генераторы номеров                                      |
| `Hierarchy`       | Иерархические свойства                                                      |
| `Historizable`    | Хранение истории изменений                                                  |
| `Geo`             | Гео-данные и операции                                                       |
| `Printer` / `QZTray` / `Sound` / `Com` | Доступ к клиентскому оборудованию                              |
| `ProcessMonitor` / `Profiler` | Управление и мониторинг сервера                                |
| `RabbitMQ` / `WebSocket` / `messengers` | Внешние транспорты                                       |
| `SQLUtils`        | Сервисные операции над БД                                                   |
| `DefaultData`     | Начальные демо-данные                                                       |
| `Schedule`        | Календарь и рабочие интервалы                                               |

Полный перечень определяется содержимым `server/src/main/lsfusion/{system,utils}/` в репозитории платформы. У модулей, для которых есть отдельная статья, ссылка ведёт туда.
