Пользовательское представление, написанное на JavaScript, взаимодействует с формой через объект *контроллера* — с его помощью представление задаёт текущий объект группы, изменяет значения свойств, запрашивает подсказки и вызывает действия или скрипты на сервере. Есть три вида пользовательских представлений; контроллер они получают по-разному, но все обращаются к одному и тому же контроллеру формы, методы которого описаны на этой странице.

Свойства и действия адресуются по их интеграционному имени — имени на форме (либо псевдониму / интеграционному имени `NEW` / `DELETE` кнопки), тому же имени, что использует [внешний JSON/REST API](How-to_Integration.md).

### Как представление получает контроллер

| представление | объявление | точка входа JavaScript | контроллер формы — это |
| --- | --- | --- | --- |
| [React-представление](How-to_Custom_React_views.md) | `DESIGN c { custom = 'Name'; }` | компонент, принимающий `props` (`data`, `controller`) | `props.controller` |
| [CUSTOM группа объектов](How-to_Custom_components_objects.md) | `OBJECTS g = Cls CUSTOM 'name'` | колбэки `render` / `update` | `controller.form` |
| [CUSTOM ячейка свойства](How-to_Custom_components_properties.md) | `PROPERTIES p CUSTOM 'name'` | колбэки `render` / `update` | `controller.form` |

**React-представление** отрисовывает целый пользовательский контейнер, поэтому `controller` в его `props` *и есть* контроллер формы — методы ниже вызываются прямо на нём.

**CUSTOM группа объектов** и **CUSTOM ячейка свойства** отрисовываются классическими колбэками `render(element, controller)` / `update(element, controller, ...)` — `update` дополнительно получает `list` строк группы либо `value` ячейки. Контроллер, который они получают, — *локальный*, привязанный к этой одной группе или ячейке: он добавляет нужные этим представлениям помощники — геттеры значения, текущей строки и стилей и `diff` / `clearDiff` для группы объектов, событие правки `change` для ячейки свойства — описанные в [Пользовательские компоненты (объекты)](How-to_Custom_components_objects.md) и [Пользовательские компоненты (свойства)](How-to_Custom_components_properties.md). Локальный контроллер выставляет контроллер формы как `controller.form`, поэтому форменные методы ниже вызываются через него:

```js
controller.form.changeObject('customer', row);
const total = await controller.form.exec('recalc', orderId);
```

Строки, которые получает классическое представление, несут те же `key` и `objects`, что и строки React (см. [Идентификация строки](#row-identity-contract)), поэтому форменные методы принимают их без изменений. Локальный `changeProperty` группы объектов также *делегирует*: для свойства, которое не является её собственной колонкой, вызов передаётся контроллеру формы и разрешается в пределах всей формы, так что представление может изменить свойство, которое не отображает.

Четвёртая точка входа — действие [`INTERNAL CLIENT`](../language/INTERNAL_operator.md): привязанная к нему JavaScript-функция получает контроллер формы аргументом после параметров вызова.

### Контроллер формы

Эти методы одинаковы везде, где доступен контроллер формы, — напрямую как `props.controller` либо как `controller.form`. Необязательные аргументы — в скобках.

| метод | что делает | возвращает |
| --- | --- | --- |
| `changeObject(groupSID, object)` | задаёт текущий объект группы | — |
| `changeProperty(property[, object][, value])` | задаёт значение либо выполняет действие — на текущем объекте или на переданной строке | — |
| `changeProperties(properties, objects, values[, groupSIDs])` | несколько вызовов `changeProperty` из параллельных массивов | — |
| `getPropertyValues(property[, object], value[, mode], ok[, fail][, count])` | ограниченный список подсказок с сервера | — (через `ok`) |
| `exec(action, ...params)` | выполняет именованное действие | `Promise` |
| `eval(script, ...params)` | выполняет lsf-скрипт с типизированным `run` | `Promise` |
| `evalAction(script, ...params)` | выполняет тело действия (параметры `$1`, `$2`, …) | `Promise` |
| `change(property, ...keyParams, value)` | задаёт глобальное свойство | `Promise` |

Изменяющие методы (`changeObject` / `changeProperty` / `changeProperties`) ничего не возвращают — новое состояние приходит со следующим обновлением формы; вызывающие сервер методы (`exec` / `eval` / `evalAction` / `change`) возвращают `Promise`. Когда интеграционное имя свойства не уникально в пределах формы, его привязывают к группе через `groupSID`: четвёртым позиционным аргументом `changeProperty` (`changeProperty(property, object, value, groupSID)`), завершающим аргументом после `count` в `getPropertyValues` либо массивом `groupSIDs` в `changeProperties`.

#### Изменение текущего объекта и значений свойств

`changeObject(groupSID, object)` задаёт текущий объект группы `groupSID`. Здесь `object` — строка данных этой группы либо непосредственный дескриптор `objects` (см. [правила идентификации](#row-identity-contract) ниже), но не голый `row.key`.

`changeProperty(property, value)` изменяет `property` для текущего объекта группы. Чтобы адресовать конкретную строку, она передаётся посередине: `changeProperty(property, object, value)`, где `object` — строка данных либо непосредственный дескриптор. Когда `property` — действие (или свойство без изменяемого значения), значение опускается: `changeProperty('edit')` выполняет его на текущем объекте, `changeProperty('edit', object)` — на переданной строке.

```js
function orderView(props) {
    const controller = props.controller;
    return (
        <div>
            <button onClick={() => controller.changeProperty('note', 'checked')}>Отметить</button>
            {props.data.o.list.map(row =>
                <div key={row.key} onClick={() => controller.changeObject('o', row)}>
                    {row.number}
                </div>)}
        </div>
    );
}
```

В форме `changeProperty(property, X)` с двумя аргументами платформа определяет, значение `X` или строка: если свойство принимает значение и `X` разрешается в строку (строка данных либо непосредственный дескриптор), то `X` читается как строка и вызов выполняется на ней; иначе `X` — это значение, и изменяется текущий объект.

`changeProperties(properties, objects, values)` применяет несколько изменений сразу из параллельных массивов — `properties[i]` изменяется на `values[i]` для `objects[i]` (элемент может быть `null` для текущего объекта). Необязательный четвёртый массив `groupSIDs` привязывает каждое свойство к группе, когда его интеграционное имя не уникально в пределах формы.

```js
controller.changeProperties(['note', 'qty'], [null, row], ['checked', 5]);
```

Групповой объект встроенного примитивного класса — например, навигатор `DATE` — перемещается DATA-свойством, от которого зависит его `FILTERS`, а не записью значения объекта: объект примитивного класса *и есть* своё значение, хранить на нём нечего. Чтобы перевести группу к произвольному значению, измените это свойство-фильтр — `changeProperty('dateFrom', d)` с настоящим JS-`Date` либо через действие — и группа последует за своими `FILTERS`; чтобы выбрать уже отображаемое значение, вызовите `changeObject` со строкой из `props.data.<g>.list` (она несёт дескриптор `objects`). Две тихие ловушки: запись собственного значения объекта группу не перемещает (хранить нечего), а `changeProperty` приводит значение-дату кастом без проверки в рантайме — поэтому аргумент не-`Date` (строка из date-инпута, timestamp) молча преобразуется в `null` или мусор, без ошибки. Передавайте настоящий `Date`.

#### Запрос значений

`getPropertyValues` запрашивает у сервера ограниченный список подсказок для свойства. Результат передаётся в обработчик `ok` в виде `{ data: [ { displayString, rawString, objects }, ... ], more }`; `more` равно `true`, когда список усечён, поэтому это список подсказок, а не полный `SELECT DISTINCT`.

```js
controller.getPropertyValues(property[, object], value[, mode], ok, fail[, count[, groupSID]]);
```

- `value` — вводимая строка запроса для сопоставления.
- `object` — необязательная строка (строка данных либо непосредственный дескриптор), ограничивающая запрос этой строкой; опускается для текущего объекта.
- `mode` — одно из:

  | `mode` | результат | `item.objects` |
  | --- | --- | --- |
  | `'objects'` (по умолчанию) | подходящие `OBJECTS` свойства — выбор объекта | непосредственный дескриптор `objects` этого объекта |
  | `'values'` | различные значения свойства | `null` (используйте `displayString` / `rawString`) |
  | `'change'` | подсказки времени правки свойства | зависит от свойства |

  `'change'` отражает то, как свойство *редактируется* — пользовательский список `INPUT`, ограничение `notNull` или пользовательское действие изменения, — а не уже имеющиеся различные значения. Для свойства, которое в этом контексте изменить нельзя (например, только для чтения или вычисляемое без действия изменения), возвращается пустой список, тогда как `'values'` всё равно вернёт его различные значения.

- `ok(result)` / `fail()` — обработчики успеха и ошибки.
- `count` — увеличивает число запрашиваемых элементов, для постраничной загрузки.
- `groupSID` — привязывает свойство к группе, когда его интеграционное имя не уникально в пределах формы (передаётся после `count`).

`item.objects` из результата режима `'objects'` можно передать напрямую обратно в `changeObject` или `changeProperty`, чтобы действовать с выбранным объектом:

```js
controller.getPropertyValues('customer', text, 'objects',
    result => result.data.forEach(item => console.log(item.displayString)),
    () => console.log('failed'));

// выбор первой подсказки как покупателя группы
controller.getPropertyValues('customer', text, 'objects', result => {
    const item = result.data[0];
    if (item) controller.changeObject('c', item.objects);
}, () => {});
```

#### Вызов сервера

`exec`, `eval`, `evalAction` и `change` выполняются на сервере и возвращают `Promise`. Они проходят ту же авторизационную проверку и преобразуют результат в значение JS так же, как серверные вызовы классического представления, — описание проверки и таблицу преобразования результата в JS см. в разделе [Вызов сервера](How-to_Custom_components_objects.md#calling-the-server).

- `exec(action, ...params)` — выполняет именованное действие; разрешается его значением `RETURN`.
- `eval(script, ...params)` — выполняет lsf-скрипт, определяющий собственное действие `run` (типизированные параметры).
- `evalAction(script, ...params)` — выполняет тело действия, обёрнутое в действие `run`, с параметрами `$1`, `$2`, ….
- `change(property, ...keyParams, value)` — изменяет глобальное свойство; последний аргумент — значение, предшествующие — ключи.

```js
const total = await controller.exec('recalc', orderId);
const doubled = await controller.eval('run(INTEGER a) { RETURN a * 2; }', 21); // 42
await controller.change('note', orderId, 'checked');
```

Вызов после закрытия формы *отклоняется* с ошибкой `Form is closed` — он никогда не зависает, — поэтому `await` на закрытой форме попадает в ветку `catch`.

### Идентификация строки {#row-identity-contract}

Метод, адресующий строку, принимает одно из:

- объект строки данных, полученный представлением (из `props` React либо из списка `update` классического представления);
- копию такой строки через spread или `Object.assign` — перечислимый дескриптор `objects` копируется вместе с ней, поэтому копия разрешается в тот же объект;
- непосредственный дескриптор `objects` — `row.objects` либо `item.objects` из результата `getPropertyValues` в режиме `'objects'`.

Голый `row.key` *не* принимается: `key` — это токен отображения / React-ключа / диффа, а не вход для разрешения. Имена полей `key`, `isCurrent` и `objects` зарезервированы на строке — свойство или колонка приложения с одним из этих интеграционных имён будет перезаписано.

Если явный аргумент-объект не разрешается ни в строку, ни в непосредственный дескриптор, платформа не подставляет молча текущую строку и не выбрасывает исключение: `changeProperty` записывает ошибку в консоль и пропускает это изменение, а `changeObject` ничего не делает.
