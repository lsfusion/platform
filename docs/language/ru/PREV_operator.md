Оператор `PREV` - создание [свойства](../paradigm/Properties.md) при помощи [оператора предыдущего значения](../paradigm/Previous_value_PREV.md).

### Синтаксис

```
PREV(propExpr)
```

### Описание

Оператор `PREV` создает свойство, возвращающее значение заданного выражения на момент начала текущей [сессии изменений](../paradigm/Change_sessions.md) — то есть значение, которое было до изменений, произошедших в этой сессии. Он всегда использует область видимости начала сессии (не переключает режим в зависимости от контекста); о том, как это соотносится с событийным режимом [операторов изменений](../paradigm/Change_operators_SET_CHANGED_etc.md), смотрите парадигмальную статью и [`Events.md#change`](../paradigm/Events.md#change). Это [контекстно-зависимый](Property_operators.md) оператор-свойство, который может встречаться внутри [выражений](Expression.md).

:::info
`PREV` это именно оператор, а не встроенное свойство с [композицией](../paradigm/Composition_JOIN.md), поэтому `PREV(f(a))` не равно `[PREV(a)](f(a))`: область видимости начала сессии распространяется на всё выражение-операнд, включая его подвыражения-аргументы. Например, в `PREV(f(g(a)))` значение `g(a)` тоже берётся на момент начала сессии; если `g` — [локальное свойство](../paradigm/Data_properties_DATA.md#local), заполненное в текущей сессии, внутри `PREV` оно вернёт `NULL`, и всё выражение тоже вернёт `NULL`. Чтобы прочитать предыдущее значение свойства при аргументах, вычисленных в текущей сессии, оберните `PREV` в отдельное свойство: при `prevF(x) = PREV(f(x))` вызов `prevF(g(a))` вычисляет `g(a)` в текущей сессии, и только `f` читается в состоянии начала сессии.
:::

### Параметры

- `propExpr`

    [Выражение](Expression.md), для которого возвращается предыдущее значение. Оно должно задавать свойство; отдельный параметр не может быть операндом.

### Примеры

```lsf
CLASS A;
f = DATA INTEGER (A);
// выдает по очереди все изменения f(a) в сессии
messageFChanges  {
    FOR CHANGED(f(A a)) DO
        MESSAGE 'В этой сессии f(a) изменилось с ' + PREV(f(a)) + ' на ' + f(a);
}

CLASS Document;
date = DATA DATE (Document);

CLASS Article;
price = DATA NUMERIC[14,2] (Document, Article);
// записать в цену документа, последнюю использованную цену в БД
// PREV важен чтобы не учитывалась цены введенные в этом документе
// это особенно важно, если последняя использованная цена будет материализованной, тогда платформа сможет просто считать это значение из таблицы
setPrice  {
    price(Document d, Article a) <- PREV((GROUP LAST price(d, a) ORDER date(d), d));
}
```
